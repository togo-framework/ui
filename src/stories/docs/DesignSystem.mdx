# Design tokens

Every component reads **semantic CSS variables** (defined in `styles.css`), never raw
colors. This is what makes theming global. The tokens are HSL-channel values exposed as
Tailwind v4 color utilities.

## Radius & type

- `--radius` drives `rounded-sm/md/lg`. Default `0.5rem`.
- Font: **Sora** (`--togo-font-body`), Latin display + body, weights 300–800.

## Rules for using the kit

1. **Never hard-code colors** in app code that wraps these components — use the same
   tokens (`bg-card`, `text-muted-foreground`, …) so your UI flips with the theme.
2. The only "always-the-same" colors are intentional: modal **scrims** (`bg-black/80`) and
   logo chips. Everything else is token-driven.

Use the **Theme** and **Direction** toolbar switches above to see tokens respond.
